Payment Templates Service

Payment Template Endpoints

  • GET /v1/template/list

    • request:
    • params: smAvailability boolean (not required)
    • response: See payments service for possible payment responses

  • GET /v1/template/{templateId}

    • response: See payments service for possible payment responses

  • POST /v1/template

    • request
    • Content-Type: multipart/form-data
      • template: payment responses
    • response: See payments service for possible payment responses

  • POST /v1/template/{paymentId}

    • request
    • Content-Type: multipart/form-data
      • template: payment responses
    • response: See payments service for possible payment responses

  • PUT /v1/template

    • request
    • Content-Type: multipart/form-data
      • template: payment responses
      • picture: <1Mb file type jpeg, jpg, png
    • response: See payments service for possible payment responses

  • DELETE /v1/template/{templateId}

payments extra fields
  Long templateId;
  String name;
  String beneficiaryName;
  Integer paymentOrderNo;
  String pictureUrl;
  Boolean smAvailability;
  Boolean favorite;
