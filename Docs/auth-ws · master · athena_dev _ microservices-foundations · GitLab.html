Authentication Service

Login, user info and logout endpoints are used by CXP. Their signatures must not change.

Login

  • POST /auth/login
    • headers:
      • Content-Type: application/x-www-form-urlencoded
      • Accept: text/html
    • parameters: username, password
    • response: token string

User info

  • GET /auth/userInfo
    • headers:
      • Authorization: Bearer
    • response:
      • Status 200 json { "username": "<username_str>", "enabled": "<true/false>", "credentialsNonExpired": "<true/false>", "accountNonLocked": "<true/false>", "accountNonExpired": "<true/false>", "authorities": [{ "authority": "ROLE_group_<group_name1>(<role_name>)" },{ "authority": "ROLE_group_<group_name2>(<role_name>)" }] }
      • Status 401 if provided token is not valid
    • authority format:
      • ROLE_group_{group_name}({role_name})
      • : group name string
      • : one of: ADMIN, MANAGER,USER,ANONYMOUS,or SYS2SYS

Logout

  • POST /auth/logout
    • headers:
      • Authorization: Bearer
    • response:
      • Status 200
      • Status 401 if token is invalid or already logged out

Check token

  • GET /auth/check_token
    • headers:
      • Authorization: Bearer
    • response:
      • Status 200
      • Status 401 if token is invalid

Renew token

  • GET /auth/renew_token
    • headers:
      • Authorization: Bearer
    • response: token string
      • Status 200
      • Status 401 if token is invalid
